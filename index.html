1955. 统计特殊子序列的数目

题目大意 :

  • 比方说，[0,1,2] 和 [0,0,1,1,1,2] 是特殊序列。
  • 相反，[2,1,0] ，[1] 和 [0,1,2,0] 就不是特殊序列。

给你一个数组 nums （仅 包含整数 0，1 和 2），请你返回 不同特殊子序列的数目 。由于答案可能很大，请你将它对 109 + 7 取余 后返回。

子数组是数组中的一个连续元素序列。

示例 1：

提示：

  • 1 <= nums.length <= 10^5
  • 1 <= nums[i] <= 10^6
  • 1 <= k <= 10^5

分析

首先肯定是找到数组中的最大值，可以使用函数max_element(begin, end)函数找，返回值是迭代器，怎么来找子数组的数量呢，使用滑动窗口

用cntMax记录窗口中最大值的数量，首先将右指针right右移，直到cntMax == k时，移动左指针，直到cntMax < k时，此时left，就表示满足条件的子数组，用res进行累加然后重复上述操作，直到没有满足条件的为止

这样选是没有重复的，因为是从左往右有顺序的进行选，每次最右边的最大值位置都是不一样的

1463. 摘樱桃 II

难度: 困难

题目：

给你一个 rows x cols 的矩阵 grid 来表示一块樱桃地。 grid 中每个格子的数字表示你能获得的樱桃数目。

你有两个机器人帮你收集樱桃，机器人 1 从左上角格子 (0,0) 出发，机器人 2 从右上角格子 (0, cols-1) 出发。

请你按照如下规则，返回两个机器人能收集的最多樱桃数目：

  • 从格子 (i,j) 出发，机器人可以移动到格子 (i+1, j-1)，(i+1, j) 或者 (i+1, j+1) 。
  • 当一个机器人经过某个格子时，它会把该格子内所有的樱桃都摘走，然后这个位置会变成空格子，即没有樱桃的格子。
  • 当两个机器人同时到达同一个格子时，它们中只有一个可以摘到樱桃。
  • 两个机器人在任意时刻都不能移动到 grid 外面。
  • 两个机器人最后都要到达 grid 最底下一行。

示例 2：

数据范围:

  • rows == grid.length
  • cols == grid[i].length
  • 2 <= rows, cols <= 70
  • 0 <= grid[i][j] <= 100

分析

这题显然是一道动态规划题，可以先思考记忆化搜素怎么实现，我们可以假设两个机器人是同时移动的，我们要记录他们的坐标，由于他们是同时移动的，那么横坐标是相同的，所以我们可以定义dfs(i, j, k)表示机器人1从(i, j)出发，机器人2从(i, k)出发，可以得到的最多樱桃数量，根据题目的要求，每次机器人移动有9总情况，对这九种情况取max即可，对于同一个点，也就是j == k的时候，只需要记录依次即可答案是dfs(0, 0, m - 1)

代码实现

857. 雇佣 K 名工人的最低成本

难度： 困难

题目大意：

有 n 名工人。 给定两个数组 quality 和 wage ，其中，quality[i] 表示第 i 名工人的工作质量，其最低期望工资为 wage[i] 。

现在我们想雇佣 k 名工人组成一个工资组。在雇佣 一组 k 名工人时，我们必须按照下述规则向他们支付工资：

  1. 对工资组中的每名工人，应当按其工作质量与同组其他工人的工作质量的比例来支付工资。
  2. 工资组中的每名工人至少应当得到他们的最低期望工资。

给定整数 k ，返回 组成满足上述条件的付费群体所需的最小金额 。在实际答案的 10-5 以内的答案将被接受。

示例 1：

提示：

  • n == quality.length == wage.length
  • 1 <= k <= n <= 10^4
  • 1 <= quality[i], wage[i] <= 10^4

分析

提示1

定理：在最优发工资方案下，至少有一名工人，发给他的工资恰好等于他的最低期望工资。

证明：反证法。假设不存在这样的工人，那么发给每名工人的工资都高于其最低期望工资，我们可以把每人的工资都下调，从而得到更优的方案，矛盾。

如何快速地求出这些工人？

2462. 雇佣 K 位工人的总代价

难度： 中等

题目大意：

给你一个下标从 0 开始的整数数组 costs ，其中 costs[i] 是雇佣第 i 位工人的代价。

同时给你两个整数 k 和 candidates 。我们想根据以下规则恰好雇佣 k 位工人：

  • 总共进行 k 轮雇佣，且每一轮恰好雇佣一位工人。

  • 在每一轮雇佣中，从最前面candidates和最后面 candidates人中选出代价最小的一位工人，如果有多位代价相同且最小的工人，选择下标更小的一位工人。

    • 比方说，costs = [3,2,7,7,1,2] 且 candidates = 2 ，第一轮雇佣中，我们选择第 4 位工人，因为他的代价最小 [3,2,7,7,1,2] 。
    • 第二轮雇佣，我们选择第 1 位工人，因为他们的代价与第 4 位工人一样都是最小代价，而且下标更小，[3,2,7,7,2] 。注意每一轮雇佣后，剩余工人的下标可能会发生变化。

  • 如果剩余员工数目不足 candidates 人，那么下一轮雇佣他们中代价最小的一人，如果有多位代价相同且最小的工人，选择下标更小的一位工人。

  • 一位工人只能被选择一次。

返回雇佣恰好 k 位工人的总代价。

示例 1：

提示：

  • 1 <= costs.length <= 10^5
  • 1 <= costs[i] <= 10^5
  • 1 <= k, candidates <= costs.length

分析

直接模拟即可，从最前面candidates和最后面 candidates人中选出代价最小的一位工人，前面选择一个最小的，后面选择一个最小的，两个进行比较，如果两个不相等，那么肯定是选择小的那么，否则如果是相等，那么肯定选择小表小的那个，也就是前面那个，这里可以使用两个优先队列来做prefix和sufix，选完一个之后，要对堆进行更新，可以使用两个变量l和r表示如果prefix pop了，那么就要将costs[l]，加到堆中，否则就是sufix pop了,那么就要将costs[r]，知道l > r的时候说明已经没有多余的候选人了

有一个特殊情况，就是如果n < 2 * candidate，那么直接所有人都被选中，那么我们只需要排个序，选择前k个即可
